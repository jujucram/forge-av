# Contributing to ForgeAV

Merci de votre intérêt pour contribuer à ForgeAV.

## Processus de contribution
1. Ouvrez une issue pour discuter des changements importants ou des nouvelles fonctionnalités.
2. Créez une branche dédiée à votre modification.
3. Soumettez une pull request propre et documentée.

## Branches
- `main` contient la version stable
- Créez une branche de fonctionnalité ou de correction de bug à partir de `main`

## Bonnes pratiques
- Gardez le code lisible et commenté
- Utilisez des noms de variables clairs
- Ne modifiez pas les dossiers `database/` sans justification
- Testez les changements avant de soumettre

## Tests
- Vérifiez les cas de scan de dossier
- Vérifiez les cas de scan de processus
- Vérifiez le fonctionnement du mode multithread
- Vérifiez la quarantaine et les logs

## Communication
- Décrivez précisément le problème ou la fonctionnalité dans l’issue
- Incluez des captures d’écran ou des exemples quand c’est utile
- Répondez aux commentaires de revue rapidement
